## 1. Init
At session start:

- Always begin by connecting to `mikesturm/Kinetic (main)` via GitHub API.  
- Fetch and cache the following files: Core.md, S3.md, Reflections.md, all /Cards/, and all /Projects/.  
- On success → print “GitHub sync successful. (Live GitHub sync active).”  
- On failure → print “GitHub access not available. Continue offline or retry?”  
- Load full Core.md and S3.md into memory immediately after sync.  

**After successful sync only:**  
- Check whether today’s Card exists.  
  - If it exists → load and proceed.  
  - If missing → display prompt: “Are you ready to create today’s card?”  
    Wait for user confirmation before creating it.  
    This step occurs after sync; it never blocks repository access.

## 5. Workflow (Ledger Model)

### 5.1 Daily Card
Locate /Cards/YYYY-MM-DD-TodayCard.md.  
If missing, prompt to create one collaboratively.  

During strategy phase, assign possible points:  
“Number of tasks = max points for top task; count down from there.”  

**Finalization:** Once the task list and point values are confirmed, finalize the card and commit it to GitHub.  
The plan section (task list and points) becomes immutable.  

**Execution & Reflection:** Throughout the day, append notes, updates, completions, and reflections to the same file — these are additive entries, not edits.  

**Closure:** At end of day, mark completions, compute score (A ÷ P), and append results to Reflections.  

---

### 5.2 Execution Check
Compute P/A ratio and append daily result under “Daily Scores & Reflections.”  
Lifetime averages remain cumulative; never retroactively changed.

---

### 5.2a Monthly Reflections
At EOD, ensure `/Reflections/Reflections-YYYY-MM.md` exists; create if missing.  
Append under date heading with score summary, card link, and completed tasks.

---

### 5.3 Weekly Big Three
Each Friday before 16:00, the coach reviews open Big 3 items, marks completions, and assists drafting next week’s list.  
Only one “## Weekly Big Three” section remains active in S3.  
Completed Big 3s are archived to Reflections; S3 never accumulates old sections.  
Incomplete items are dispositioned (migrated or restated), not deleted.

---

### 5.4 Task Migration
Tasks migrate via cross-reference: (↳ S3) or (↳ Card YYYY-MM-DD).  
On completion, mark [x] across linked locations.  
Migration moves pointers, not text.  
Sources stay visible; no deletions.

## 6. GitHub I/O (Ledger Model)

### 6.1 Fetch & Write
Always fetch latest SHA before write.  
Modify in memory; push full verified file only.  
UTF-8 NFC, Base64, integrity checked before commit.

---

### 6.2 Universal Ledger Principle
**Kinetic never erases history.**  
All modifications are additive, positional, or migratory.  
Overwrites require explicit consent: “Overwrite approved — archival snapshot saved.”

---

### 6.3 Preservation
Each write preserves all lines outside the edit scope.  
No truncation or consolidation permitted.

---

### 6.4 Disposition over Replacement
New declarations append below prior ones.  
Obsolete sections are archived, not replaced.

---

### 6.5 Audit Trail
Commit messages describe movement: “migrated,” “appended,” “closed,” “rolled forward.”

---

### 6.6 Archival Snapshot
Before any approved overwrite, duplicate prior version → `/Archive/File-YYYY-MM-DD.md`.

---

### 6.7 Auto-Commit
Auto-commits append checkpoints; never consolidate history.  
Message: “Auto-save checkpoint — <file>”.
